# Требования к новым сервисам "Умный дом" для PropDevelopment 
## Требования к безопасности 
### Защита персональных данных 
- Обеспечить шифрование биометрических данных (фото лиц) и данных об автомобилях как при передаче, так и при хранении в соответствии с 152-ФЗ
- Реализовать механизм аниспуфинга (anti-spoofing) для предотвращения обмана системы распознавания лиц
- Внедрить защиту от подделки автомобильных номеров в системе распознавания шлагбаума

### Контроль доступа к данным 
- Гарантировать, что данные одного пользователя не будут отображаться другому пользователю (устранение текущей проблемы, описанной в жалобах клиентов)
- Ограничить доступ к данным о распознавании лиц и номеров только авторизованным пользователям и системам
- Внедрить разделение данных по жилым комплексам, чтобы предотвратить ситуацию, когда партнер одной управляющей компании может получить доступ к данным другой компании

### Безопасность API 
- Определить четкие контракты API с партнером, ограничивающие передаваемые данные только необходимыми
- Внедрить систему мониторинга и аудита всех API-вызовов к партнерским системам
- Установить лимиты на количество запросов для предотвращения злоупотребления системой

### Соответствие законодательству 
- Получить явное согласие пользователей на обработку биометрических данных
- Реализовать механизм удаления биометрических данных по запросу пользователя
- Обеспечить возможность экспорта данных пользователя в соответствии с требованиями законодательства

## Протоколы аутентификации и авторизации 
### Единая система аутентификации 
- Интегрировать новые сервисы с существующей системой tenant-core-app через протокол OpenID Connect
- Использовать существующую систему аутентификации собственников без создания новых точек регистрации
- Внедрить единый механизм управления сессиями для всех сервисов
         

### Многофакторная аутентификация 
- Для критически важных операций (например, изменение настроек доступа) использовать двухфакторную аутентификацию
- Поддерживать биометрическую аутентификацию на мобильном устройстве (Touch ID/Face ID)
         

### Ролевая модель авторизации (RBAC) 
- Определить четкие роли: собственник квартиры, гость, администратор ЖК, управляющая компания
- Реализовать проверку доступа на уровне объекта (Object-level permissions) - пользователь может управлять только тем домофоном/шлагбаумом, который относится к его жилому комплексу
- Внедрить механизм проверки прав при каждом запросе к партнерской системе
         

### Безопасная авторизация для партнерских систем 
- Использовать mutual TLS (mTLS) для взаимной аутентификации между системами PropDevelopment и партнером
- Ограничить IP-адреса партнерских систем, с которых разрешены запросы
- Внедрить механизм временных токенов доступа для партнерских систем с коротким сроком жизни
         
     

## Организация взаимодействия между системами 
### Архитектура интеграции 
- Все запросы к партнерским системам должны проходить через API Gateway PropDevelopment, а не напрямую из мобильного приложения
- Интегрировать новые сервисы с доменом "Группа сервисов ЖКУ" через существующий tenant-core-app
- Внедрить шину данных (ESB) или использовать API Management для централизованного управления интеграциями
         

### Протоколы взаимодействия 
- Использовать HTTPS с TLS 1.2+ для всех внешних соединений
- Для передачи данных использовать JSON с четко определенными схемами и версионированием
- Реализовать механизм idempotency keys для критически важных операций
         

### Безопасная обработка данных 
- Внедрить механизм преобразования данных (data masking) для конфиденциальных данных при передаче
- Использовать JWT с цифровой подписью для передачи информации о пользователе между системами
- Реализовать шифрование данных на уровне приложения для биометрических данных
         

### Мониторинг и аудит 
- Централизованное логирование всех взаимодействий с партнерскими системами
- Внедрить систему обнаружения аномалий в запросах к системам распознавания
- Регулярный аудит безопасности интеграций как часть комплексного аудита компании
         

### Отказоустойчивость и обработка ошибок 
- Реализовать механизм circuit breaker для предотвращения каскадных отказов
- Обеспечить работу системы в режиме ограниченной функциональности при недоступности партнерской платформы
- Внедрить механизм повторных попыток с экспоненциальной задержкой для временных сбоев
         

### Согласованность данных 
- Определить tenant-core-app как источник истины для данных о собственниках
- Внедрить механизм синхронизации данных между системами с проверкой целостности
- Реализовать механизм компенсирующих транзакций для отката операций в случае ошибок
